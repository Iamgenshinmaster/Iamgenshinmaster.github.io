动态采样算法

扩展Olken算法(Extended Olken)

快速采样，高效处理

核心思想

结合传统Olken算法，使用主键最大频率信息与AGM边界优化权重上界估计，降低拒绝率

技术特点

  • 使用阈值划分高频/低频数据
  • 高频部分使用AGM边界方法
  • 低频部分使用传统Olken方法
  • 无需完整数据分布信息
  • 计算速度快，资源消耗低

适用场景

简单自然链式连接、中小规模数据量

精确权重算法(Exact Weight)

高精度采样，零拒绝率

核心思想

通过动态规划和逆向迭代的方法计算精确权重，以时间成本换取高精度的动态采样

技术特点

  • 零拒绝率，采样效率最高
  • 完美的均匀性
  • 从后向前逆向计算权重
  • 支持复杂查询情况
  • 结果精度最高

适用场景

大批量采样、精度要求高、复杂查询场景

在线探索算法(Online Exploration)

平衡效率与精度

核心思想

融合随机游走与动态规划，利用置信区间估计动态调整权重上界，平衡效率与准确性

技术特点

  • 随机游走生成样本路径
  • 置信区间估计权重上界
  • 动态规划补充低频数据
  • 自适应平衡效率与精度
  • 减少数据倾斜影响

适用场景

超大规模数据、需要平衡效率与精度、数据分布复杂场景

查询语句设置

查询 T

三角连接查询

查询描述

发现三个用户之间的三角形关联关系，涉及一个流行用户和两个普通用户，形成一个闭合的互动三角

SQL 语句

SELECT * 
FROM Popular_user A, Twitter_user B, Twitter_user C
    WHERE A. dst = B. src 
    AND B. dst = C. src 
    AND C. dst = A. src;

关系说明

流行用户 ×1 普通用户 ×2 三角关系

使用示例

流行用户A关注普通用户B，B关注普通用户C，C关注A

查询 S

四角连接查询

查询描述

追踪多级传播路径，涉及四个用户的连接，由一个流行用户和三个普通用户组成

SQL 语句

SELECT *
FROM Popular_user A, Twitter_user B, 
    Twitter_user C, Twitter_user D
    WHERE A. dst = B. src
    AND B.dst = C.src
    AND C.dst = D.src
    AND D.dst = A.src;

关系说明

流行用户 ×1 普通用户 ×3 四级传播

使用示例

A发布一条帖子，B转发，C评论了B的转发，D点赞了C的评论

查询 F

雪花连接查询

查询描述

模拟雪花模式，涉及层级化的连接，由两个流行用户和两个普通用户构成

SQL 语句

SELECT *
FROM Popular_user A, Twitter_user B,
    Twitter_user C, Popular_user D
WHERE A. src = B. src
AND C.dst = A.src
AND C. src = D. src;

关系说明

流行用户 ×2 普通用户 ×2 层级连接

使用场景

流行用户A的关注者C，发表了一篇文章，流行用户B的关注者D点赞了这篇文章

实验数据集

斯坦福SNAP数据集

推特社交关系数据

数据集描述

来自斯坦福SNAP提供的推特用户关系数据集，包含用户之间的关注、互动等关系

关键数据指标

  • 用户数量: 81,306名
  • 关系数量: 1,768,149个
  • 最大弱连通分量: 包含全部用户
  • 最大强连通分量: 68,413用户 (占84%)
  • 数据规模: 百万级别

应用场景

用于构建多表连接场景（三角、四角、雪花连接），验证动态采样算法的性能

TPC-H数据集

决策支持基准数据

数据集描述

由交易处理效能委员会(Transaction Processing Performance Council)制定的基准程序，由一套面向业务的临时查询和并发数据修改组成

关键数据指标

  • 数据规模: 1GB
  • 包含关系: 8个基本表
  • 测试查询: 22个复杂查询
  • 评价指标: 查询响应时间
  • 应用场景: 全连接对比实验

查询示例

SELECT c_custkey, o_orderkey, l_linenumber 
FROM customer, orders, lineitem 
WHERE c_custkey = o_custkey 
  AND l_orderkey = o_orderkey;

实验结果

优化前后对比结果

图1: 优化前后时间对比 (TPC-H数据集)
本图展示了三种优化算法(EO, EW, OE)与全连接方法在不同样本规模下的时间对比。全连接方法在样本规模增加时时间消耗显著增长，而三种优化方法在样本规模不大时均优于全连接方法

总体运行时间

图2: 查询T(三角连接)的总体运行时间

关键发现：EO算法在小样本量时表现最佳，但随着样本量增加效率下降；EW算法初始准备时间长但后期采样效率高；OE算法表现稳定

图3: 查询S(四角连接)的总体运行时间

关键发现：四角连接复杂度增加，EO算法效率下降明显，EW算法在大样本量下保持高效，OE算法在效率与精度间取得平衡。

图4: 查询F(雪花连接)的总体运行时间

关键发现：雪花连接场景下，EO算法采样效率最低耗时长，EW算法和OE算法综合表现几乎一样优秀

首次抽样准备时间

图5: 三种算法在不同查询下的首次抽样准备时间
三种方法所需的准备时间EO是最少的，OE方法次之，EW方法最高： EO方法在计算W值的过程中不需要额外的准备时间，因此最快。 OE方法具体的准备时间与随机游走的次数有关，在随机游走过程中需要进行下一个结点的选择和概率的记录，在所有随机游走结束后才开始计算上界估计值，因此所花时间较长。 EW方法需要对每一个节点使用动态规划精确计算上界估计值，因此所需时间最长。

分析与展望

实验结果分析

算法性能总结

扩展Olken算法 (EO)

  • 优势: 初始化最快，资源消耗低，适用于实时性要求高场景
  • 劣势: 大规模采样时拒绝率高，效率随样本增加而下降
  • 适用场景: 简单自然连接、中小规模数据

精确权重算法 (EW)

  • 优势: 零拒绝率，采样效率最高，精度最佳
  • 劣势: 初始化成本高，内存需求大，不适合动态环境
  • 适用场景: 大批量采样、精度要求高、复杂查询场景

在线探索算法 (OE)

  • 优势: 平衡效率与精度，动态适应性强
  • 劣势: 实现复杂，参数设置敏感，工程难度大
  • 适用场景: 超大规模数据、数据分布复杂、需要平衡的场景
实验结论

动态采样技术能有效提升基数估计精度，但算法选择需权衡效率与精度：

  • 对于简单查询和小规模数据，EO算法是最佳选择
  • 对于复杂查询和高精度要求，EW算法表现最优
  • 对于大规模数据和需要平衡的场景，OE算法最为合适
